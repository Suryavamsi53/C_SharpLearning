  • 1. Classes and Objects:

    Class: A blueprint for creating objects. Object: An instance of a class.

  • 2. Inheritance:

    Base and Derived Classes: Establishing a relationship where a class inherits properties and behaviors from another class.

    Method Overriding: Modifying the implementation of a method in a derived class.

    Abstract Classes and Methods: Creating abstract classes with abstract methods that must be implemented by derived classes.

    Sealed Classes: Preventing further inheritance of a class.

  • 3. Polymorphism:

    Method Overloading: Defining multiple methods with the same name but different parameters. Interface and Implementation: Implementing interfaces to achieve polymorphic behavior. Operator Overloading: Defining custom behaviors for operators.

  • 4. Encapsulation:

    Access Modifiers (public, private, protected, internal): Controlling access to class members.

    Properties: Providing controlled access to private fields.

  • 5. Interfaces:

    Declaring and Implementing Interfaces: Defining contracts that classes must adhere to.

    Multiple Interfaces: A class can implement more than one interface.

  • 6. Structs and Enums:

    Structs: Value types similar to classes but typically used for lightweight objects. Enums: Named constants representing a set of named values.

  • 7. Delegates:

    Declaring and Using Delegates: Representing references to methods. Multicast Delegates: Combining and invoking multiple methods.

  • 8. Events:

    Declaring and Raising Events: A mechanism for communication between objects. Event Handlers: Methods that respond to events.

  • 9. Exceptions and Error Handling:

    try, catch, finally Blocks: Handling and recovering from runtime errors. Custom Exceptions: Creating user-defined exception classes.

  • 10. Nullable Types:

    Nullable Value Types: Allowing value types to have a null value.

  • 11. Lambda Expressions and LINQ:

    Anonymous Methods: Inline, unnamed methods. Lambda Expressions: Concise syntax for writing anonymous methods. LINQ (Language-Integrated Query): Querying collections using a SQL-like syntax.

  • 12. Asynchronous Programming:

    Async and Await Keywords: Simplifying asynchronous programming.

    Task and Task Types: Representing asynchronous operations.

  • 13. Reflection and Attributes:

    Reflection: Inspecting and interacting with types at runtime.

    Attributes: Providing metadata to classes, methods, and properties.

  • 14. Dynamic Language Runtime (DLR):

    Dynamic Keyword: Deferring type checking until runtime.

    ExpandoObject and Dynamic Types: Creating objects with dynamic members.

  • 15. Pattern Matching:

    Switch Expressions: A more powerful version of the traditional switch statement.

    Type Patterns: Matching and deconstructing types in patterns.

  • 16. File I/O and Serialization:

    Reading and Writing to Files: Interacting with the file system.

    Serialization and Deserialization: Converting objects to and from a format for storage or transmission.

  • 17. Advanced Collections:

    Queues, Stacks, and Specialized Collections: Additional data structures for specific use cases.

    Collection Interfaces (IEnumerable, ICollection, IList): Common interfaces for interacting with collections.

  • 18. Garbage Collection:

    Memory Management: Automatic memory management in C#.

    IDisposable Interface and Using Statement: Manual resource management.

  • 19. Dependency Injection:

    IoC Containers: Inversion of Control for managing object dependencies.

    Dependency Injection Patterns: Injecting dependencies into classes rather than having classes create their dependencies.

  • 20. Unit Testing:

    Writing and Running Unit Tests: Ensuring code correctness and reliability.

    Testing Frameworks (NUnit, MSTest): Tools for creating and running unit tests.

    These concepts form the foundation of C# programming and are crucial for building scalable, maintainable, and efficient applications

    • C# try and catch

    The try statement allows you to define a block of code to be tested for errors while it is being executed.

    The catch statement allows you to define a block of code to be executed, if an error occurs in the try block.

    The try and catch keywords come in pairs:

    Syntax

    try

    {

    // Block of code to try

    }

    catch (Exception e)

    {

    // Block of code to handle errors

    }

    Finally

    The finally statement lets you execute code, after try...catch, regardless of the result:

    Example

    try

    {

    int[] myNumbers = {1, 2, 3};

    Console.WriteLine(myNumbers[10]);

    }

    catch (Exception e)

    {

    Console.WriteLine("Something went wrong.");

    }

    finally

    {

    Console.WriteLine("The 'try catch' is finished.");

    }

    The output will be:

    Something went wrong.

    The 'try catch' is finished.

    The throw keyword

    The throw statement allows you to create a custom error.

    The throw statement is used together with an exception class. There are many exception classes available in C#:

      ArithmeticException,

      FileNotFoundException,

      IndexOutOfRangeException,

      TimeOutException, etc:

    System.Exception:

    The base class for all exceptions.

    System.ArgumentException:

    Thrown when an argument to a method is not valid.

    System.ArgumentNullException:

    Thrown when a null argument is passed to a method that doesn't accept it.

    System.ArgumentOutOfRangeException:

    Thrown when an argument value is outside the allowable range.

    System.InvalidOperationException:

    Thrown when a method call is invalid for the object's current state.

    System.NotSupportedException:

    Thrown when an operation is not supported.

    System.NullReferenceException:

    Thrown when an attempt is made to access an object that is null.

    System.IndexOutOfRangeException:

    Thrown when an array index is outside the bounds of the array.

    System.DividedByZeroException:

    Thrown when there is an attempt to divide an integer or decimal by zero.

    System.OutOfMemoryException:

    Thrown when an attempt to allocate memory fails.

    System.StackOverflowException:

    Thrown when the execution stack overflows.

    System.FormatException:

    Thrown when the format of an argument is invalid.

    System.IO.IOException:

    Base class for exceptions thrown by methods in the System.IO namespace.

    System.Threading.ThreadAbortException:

    Thrown when a thread is being aborted.

    System.Net.WebException:

    Thrown by classes in the System.Net namespace when a network error occurs.

    System.Xml.XmlException:

    Thrown for XML-related errors.

    System.Security.SecurityException:

    Thrown when a security violation is detected.

    System.Reflection.TargetInvocationException:

    Thrown by the Invoke method on the System.Reflection.MethodBase class.

    System.Runtime.InteropServices.COMException:

    Thrown for COM interop errors.

    System.Data.SqlClient.SqlException:

    Thrown for errors specific to SQL Server.

    Example

    static void checkAge(int age)

    {

    if (age < 18)

    {

    throw new ArithmeticException("Access denied - You must be at least 18 years old.");

    }

    else

    {

    Console.WriteLine("Access granted - You are old enough!");

    }

    }

    static void Main(string[] args)

    {

    checkAge(15);

    }

    The error message displayed in the program will be:

    System.ArithmeticException: 'Access denied - You must be at least 18 years old.'

    If age was 20, you would not get an exception:
